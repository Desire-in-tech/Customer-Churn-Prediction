# Customer Churn Prediction System

A full-stack ML application that predicts telecom customer churn. Enter a customer's account details and get a real-time prediction: Churn or Stay, probability percentage, risk level (Low/Medium/High), and the top SHAP-derived risk factors driving the prediction.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the Express API server (port 8080, serves `/api`)
- `pnpm --filter @workspace/frontend run dev` — run the React frontend (port 18130, serves `/`)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- Django backend: `cd backend && pip install -r requirements.txt && python manage.py runserver 8000`

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + TailwindCSS + Recharts
- API (Replit): Express 5 (serves `/api/*` routes)
- API (production): Django 4.2 + Django REST Framework (in `backend/`)
- ML: XGBoost + scikit-learn + SHAP + imbalanced-learn (SMOTE)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — API contract (source of truth)
- `lib/api-client-react/src/generated/` — generated React Query hooks
- `lib/api-zod/src/generated/` — generated Zod schemas
- `artifacts/frontend/src/` — React app (Dashboard + Predict pages)
- `artifacts/api-server/src/routes/predict.ts` — Express prediction routes
- `backend/` — Django REST API (for Render deployment)
- `backend/ml/predictor.py` — ML inference bridge (loads model.pkl + preprocessor.pkl)
- `backend/ml/model.pkl` — trained XGBoost model (generated by notebook)
- `backend/ml/preprocessor.pkl` — fitted ColumnTransformer (generated by notebook)
- `notebooks/churn_model.ipynb` — full ML pipeline (run in Google Colab or Jupyter)
- `data/telco_churn.csv` — IBM Telco dataset (download from Kaggle, see data/README.md)

## Architecture decisions

- **Dual backend**: Express API server handles Replit preview routing; Django backend is the production target for Render deployment. Both expose the same `/api/predict/` contract.
- **Rule-based fallback**: When `model.pkl` is absent, both backends use a heuristic probability function so the app demos immediately without needing a trained model.
- **SHAP at inference**: The Django predictor runs SHAP TreeExplainer at inference time to surface per-customer risk factors. Falls back to rule-based factors when SHAP fails.
- **OpenAPI-first**: All API contracts are defined in `lib/api-spec/openapi.yaml` and codegen'd to React Query hooks + Zod schemas. Never hand-write what codegen produces.

## Product

- **Dashboard** (`/`): 4 KPI cards (7,043 customers, 1,869 high-risk, avg tenure 32mo, avg charges $64.76), contract type bar chart, internet service pie chart, recent predictions table.
- **Predict Risk** (`/predict`): 19-field customer form (pre-filled with high-churn defaults for demo), animated probability ring, risk badge (Low/Medium/High), top 5 SHAP risk factors.

## Notebook workflow (Google Colab)

1. Download IBM Telco dataset → save as `data/telco_churn.csv`
2. Open `notebooks/churn_model.ipynb` in Google Colab
3. Run all cells — pipeline: EDA → cleaning → feature engineering → preprocessing → SMOTE → XGBoost → SHAP → save artifacts
4. Download `model.pkl` and `preprocessor.pkl` → place in `backend/ml/`

## Gotchas

- `TotalCharges` loads as string in the CSV — the notebook converts it to float and imputes new customers (tenure=0) with `tenure × MonthlyCharges`.
- Django `views.py` imports predictor via `backend.ml.predictor` (not relative `..ml.predictor`) because Django is run from the `backend/` directory with the project root on `PYTHONPATH`.
- The `data/telco_churn.csv` file is gitignored — add it locally before running the notebook.
- Run `pnpm --filter @workspace/api-spec run codegen` after any OpenAPI spec change before touching frontend code.
